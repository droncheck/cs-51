# Интерпретатор байткода

## Функция `execute(program)`

`execute` принимает массив чисел — плоский байткод — и интерпретирует его. Состояние машины: аккумулятор `A` (переменная в замыкании) и указатель на текущую запись в разобранном списке команд `p`.

Работа разделена на **два этапа**.

### 1. Разбор

Указатель идёт с начала `program` к концу. На каждом шаге читается код операции.

- Если код относится к команде **с аргументом** (в текущей схеме это `SET A`, `RET`, `JMP`), из потока читаются два числа подряд: код и аргумент. В `commands` добавляется пара `[код, аргумент]`, затем служебная запись с маркером `-1`, указатель в исходном массиве сдвигается на два элемента.
- Иначе команда **без аргумента**: в `commands` попадает один элемент `[код]`, указатель сдвигается на одно число.

Маркер `-1` не соответствует реальной инструкции: при исполнении он только увеличивает `p` на единицу. Он нужен, чтобы **индексы в массиве `commands` совпадали с теми адресами, которые задаёт операнд `JMP`**

Если для кода, требующего аргумент, в конце массива нет следующего числа, выбрасывается ошибка с именем инструкции (через `getInstructionName`).

### 2. Исполнение

Бесконечный цикл по `commands[p]`:

| Код       | Действие                                                                                                            |
| --------- | ------------------------------------------------------------------------------------------------------------------- |
| `-1`      | `p += 1` (пустой шаг разметки).                                                                                     |
| `SET A`   | записать аргумент в `A`, затем `p += 1`.                                                                            |
| `PRINT A` | вывести `A` в консоль, `p += 1`.                                                                                    |
| `IFN A`   | если `A === 0`, то `p += 1`; иначе `p += 2` (переход через одну ячейку — слот под маркер после предыдущей команды). |
| `RET`     | вернуть аргумент как результат функции.                                                                             |
| `DEC A`   | уменьшить `A` на 1, `p += 1`.                                                                                       |
| `JMP`     | присвоить `p` значению аргумента (безусловный переход по индексу в `commands`).                                     |

Если `p` выходит за границу `commands` до того, как сработает `RET`, выбрасывается `Program ended unexpectedly`. Неизвестный код в первой позиции записи — отдельное исключение.

Итог: **`execute` сначала нормализует поток чисел в последовательность логических команд с сохранением адресации для `JMP`, затем пошагово выполняет их до `RET` или ошибки.**

## Ошибки

- Неполная команда с аргументом в конце `program`.
- Неожиданный конец без `RET`.
- Неизвестный числовой код операции.
